# Issue Tracking Guide

## GitHub Issues Setup

This project uses **GitHub Issues** for tracking bugs, enhancements, and feature requests.

### Issue Templates Available

1. **Bug Report** - Report bugs with detailed reproduction steps
2. **Feature Request** - Suggest new features or enhancements

### How to Create an Issue

1. Go to the GitHub repository
2. Click the "Issues" tab
3. Click "New Issue"
4. Select a template (Bug Report or Feature Request)
5. Fill in the required information
6. Click "Submit new issue"

### Labels Used

- **bug**: Something isn't working
- **enhancement**: New feature or improvement
- **documentation**: Improvements or additions to documentation
- **in-progress**: Currently being worked on
- **help wanted**: Extra attention is needed
- **question**: Further information is requested

### Example Issue Comment

When working on an issue, submit a comment like:

```
Working on this issue! 

## Planned Changes
- [ ] Refactor audio analysis logic
- [ ] Add frequency band visualization
- [ ] Improve performance optimization

## Related
- Closes #[issue number]
```

### Workflow

1. **Create**: Open an issue describing the problem/feature
2. **Discuss**: Comments collaborate on the approach
3. **Plan**: Outline the solution in the issue
4. **Implement**: Create a branch and make changes
5. **Comment**: Update the issue with progress
6. **Review**: Submit a pull request referencing the issue
7. **Resolve**: Close the issue when the PR is merged

### Tips

- Use clear, descriptive titles
- Include environment information for bugs
- Add labels for easy filtering
- Reference related issues with `#[issue-number]`
- Use checklists for tracking progress
